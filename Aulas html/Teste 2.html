DOM

Document Object Model. API que representa os objetos de uma página HTML em forma de uma árvore e lê o documento como um objeto, permitindo alterar conteúdos (alterar, mover, excluir, etc) na aplicações web em tempo de execução, ou seja, sem precisar recarregar a página completamente. Através dele acessamos os elementos dentro de uma página HTML. O DOM cria um objeto diferente para cada elemento, mas vincula cada um deles ao seu elemento circundante (ou elemento pai), formando uma árvore hierárquica.

Árvore hierárquica de DOM:

GetElementById, GetElementByClass, getElementsByClassName e getElementsByTagName

Get Elemet By permite obter um elemento do documento a partir de um atributo ID, class, name ou tag já especificado para o elemento.

Tags semânticas

Quando se escree uma página HTML, utilizam-se tags semânticas para criar sessões no texto. Estas tags substituem (ainda que não obrigatoriamente) as divs, pois facilitam na compreensão da semântica do código. Sãoexemplos de tags semânticas: header (cabeçalho), nav (navegação), aside (coluna/quadro ateralizado), footer (rodapé), e article (artigo)

Tabelas: table, td, tr e th

Tag table: delimita o início e o fim da tabela.
Tag tr: define uma linha. Quantas sejam a linhas da tabela, se abrem tr´s em sequencia, uma baaixo da outra. Abre e fecha dentro da table.
Tag td: define um dado. Quantas sejam as colunas, se abrem td´s em sequência, uam abaixo da oura.Abre e fecha dentro da tr.
Atributo border: deve ser acrescentado dentro da abertura da tag table (com um valor igual a 1) para inserir as bordas da coluna.
Outro atributo de coluna é o colspan, que serve para mesclar células de coluna. Por exemplo, para mesclar três células de uma mesma coluna, se abre o atributo colspan dentro da primeira tr que fará parte da mescla de célula, e ="3".
O mesmo se aplica para linhas, com o atributo rowspan.
A tag th (tablehead) centraliza e negrita o conteúdo da célula.
