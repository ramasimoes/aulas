DOM

Document Object Model. API que representa os objetos de uma página HTML em forma de uma árvore e lê o documento como um objeto, permitindo alterar conteúdos (alterar, mover, excluir, etc) na aplicações web em tempo de execução, ou seja, sem precisar recarregar a página completamente. Através dele acessamos os elementos dentro de uma página HTML. O DOM cria um objeto diferente para cada elemento, mas vincula cada um deles ao seu elemento circundante (ou elemento pai), formando uma árvore hierárquica.

Árvore hierárquica de DOM:

GetElementById, GetElementByClass, getElementsByClassName e getElementsByTagName

Get Elemet By permite obter um elemento do documento a partir de um atributo ID, class, name ou tag já especificado para o elemento.

    Seletor ID e Seletor de classes Class

    Id(identificador>): Estiliza todos os elementos especificados com o seletor ID. É único em uma página HTML, então todos os elementos (as tags onde são aplicados) ID devem ter o mesmo valor. Precedido por uma #.
    Class: Para agrupar elementos dentro de alguma característica (atributo "classe") específica. Pode aparecer várias vezes. Se aplica em HTML inserindo o atributo class em alguma tag, e definindo, em CSS, qual será a classe, podendo receber qualquer nome. Em CSS, é precedido pelo . (ponto final).
    Hierarquia de especificidade (do menos específico mara o mais específico) e respectivos valores para cálculo de especificidade:

    1. Elemento (a tag) = 1
    2. Classe (.class), pseudo-classe e atributo = 10
    3. Identificador (#id) = 100
    4. Estilo inline, ou seja, o que foi descrito em HTML = 1000

    Em caso de empate, a últia regra definida será assumida.
    Para que o navegador NÃO considere alguma especificidade, se utiliza !important depois da definição do id/tag/class/enfim, ainda dentro do {}.

    Box Model


    • Padding: sempre transparente. è o espaçam,ento interno entre o texto e a borda da caixa onde ele se encontra.
    • Borda (border): define o tamanho da caixa. Possui as propriedades tamanho (em px), estilo e cor.
    • Margem (margin): espaçamento externo, sempre transparente. Define a distância entre a borda de duas caixas.

    Tipos de elementos da caixa:

    • De bloco: respeita as propriedades de altura e largura. ocupa todo o espaço da horizontal, e aí então a caixa gera uma nova linha. Elementos de bloco: div, h1 até h6, blockquote, ul, ol, p, form, header, table, etc.
    • De linha: tem tamanho definido pelo conteúdo, ou seja, não respeita a propriedade de altura e largura. Não pode conter elementos de bloco. Margem e espaçamento interno não possuem o mesmo efeito (comportamentos diferentes na horizontal e vertical). Exemplos: a, br, img, object, q, script, span, input, button, label, select, textarea etc.

    Display

    Define o comportamento com que os elementos HTML serão apresentados. Distingue elementos de linha dos de bloco e modifica comportamentos padrão dos elementos.


    Tipos de display:
    • Nenum: p {display: none;} o elemento existe mas não é exibido na interface.
    • Em linha: p {display: inline;} para elemento inline. Não aceita mudar tamanho da fonte.
    • Em bloco: p {display: block;}
    • Em bloco/linha: p {display:inline-block;} funciona como elemento de linha mas permite que aplique características de um elemento de bloco em algum trecho da linha. Permite aplicar um bloco dentro da linha. Aceita mudar o tamanho da fonte para o que estiver dentro do bloco.

      • Posicionamento

      Top, right, left e bottom.

      • Estático: padrão. Ignora top right left e bottom. Utilizado também para desfazer um posicionamento para algum elemento do HTML que tenha sido definido por outra regra. p {position: static;}
      • Relativo: posicionado em relaçã oà sua posição normal Não ignora top right left e bottom. Espaços deixados pelo posicionamento não serão preenchidos. Exemplo: p {position: relative};
      • Fixo: fixado na tela. Sempre ocupa o mesmo espaço, independente do conteúdo. Precisa definir pelo menos uma propriedade top right left e bottom. Não deixa espaços. Exemplo: p {position: fixed, right 0; bottom 0;}.
      • Absoluto: È absoluto a partir de um elemento relativo. Se posiciona a partir de seu ancestral. Precisa definir pelo menos uma propriedade top right left e bottom. Nã odeixa espaços, pode sobrescrever elementos. Exemplo: p {position: absolute; right 0; bottom 0;}
      • Sticky: Elemento é posicionado de acordo com a rolagem da tela. Funciona como um elemento relativo e fixo de acordo com a barra de rolagem. Não deixa espaços, pode sobrescrever elementos, preicsa de pelo menos um top right left e bottom. Exemplo: p {position: sticky; top: 0;}.

      Uma linah que pode se estender em várias linhas; o mesmo para coluna.

      Flex Container

      Flex box layout module. Alternativa pra criação de layouts, sem float e posicionamentos. Para criar um container, se usa a propriedade display. Exemplo: .container {display: flex}. Têm 3 propriedades:

      • Direção: propriedade flex-direction. Seus valores podem ser column, column-reverse, row e row-reverse. Exemplo: .container {display: flex; flex-direction: column}
      • Quebra: propriedade flex-wrapdefine como os elementos "quebram" de linha/coluna. Os valores possíveis são wrap, wrap-reverse e nowrap. Exemplo: .container {display:flex; flex-wrap: wrap}
      • Nowrap: não permite a quebra, diminuindo o tamanho dos intens.
      • Alinhamento: propriedade justify-content. Define como os elementos serão alinhados horizontalmente. Propriedades: center, flex-start, flex-end, space-around e space-between. Exemplo: .container {display: flex; justify-content: center}

      Pilhas de elementos z-index

      Elementos CSS podem ocupar um mesmo espaço (eixo x,y) e são empilhados no eixo z (tridimensional). O último elemento 'empilhado' é o que fica visível.
      Propriedade z-index: define a ordem de visualização dos elementos. Funciona só para elementos absoluto, relativo, fixo e sitcky. Quanto maior seu valor, mais a frente ele estará. Exemplo: p {z-index: -1}

      Overflow

      Propriedade que controla o comportamento dos elementos de blocos quando o conteúdo é maior que o limite do elemento.

      • visible: fica visível e ultrapassa os limites do elemento

      Git

      Ciclo de vida dos arquivos.

      Git

      É um sistema de controle de versão de arquivos. Permite que várias pessoas possam contribuir e modificar simultaneamente num mesmo projeto/arquivo, editando e criando novos arquivos e permitindo que esses novos arquivos modificados possam existir sem o risco de suas alterações serem sobrescritas.

      untracked -> unmodified -> modified -> staged

      Novo diretório

      Git init. A execução desse comando cria um novo subdiretório .git no diretório de trabalho atual. Essa ação também vai criar uma ramificação principal.

      Commit: utilizado para informar ao git que queremos gerar um novo estado com os arquivos em staged.

      Branch: Cria uma linha alternativa no tempo pra trabalhar em paralelo à branch principal.

      • Untracked: o git ainda não conhece a existência do arquivo em nenhuma versão
      • Unmodified: o arquivo tem um estado anterior registrado no git. São os arquivos que não sofreram nenhuma alteraçao em relação à última versão.
      • Modified: arquivos que sofreram alteração em relação à sua última versão.
      • Staged: arquivos que estão prontos para serem inseridos na nova versão (esse novo arquivo se chama snapshot)
      • Git status: consultar o status dos arquivos
      • Git log: alterar ou consultar commits
        Git log --graph: consultar commits visualizando as ramificações
      • Git commit -m: comenta a alteração
      • Git branch nome_branch: criar uma nova branch
      • Git checkout nome_branch: mudar de branch
      • Git merge nome_branch: unir/mergear duas branches

      Git reset:

      • Git checkout nome_arquivo: voltar o arquivo para antes da edição
      • Git diff nome_arquivo: ver modificaçõe em um arquivo
      • Git reset HEAD nome_arquivo: voltar o arquivo para o staged (como era antes de qualquer modificação)
      • Git reset:
        1. --soft id_commit_anterior
        2. --mixed id_commit_anterior
        3. --hard id_commit_anterior

      Git remote:

      • Git remote add origin url_repo: adicionar um repositório remoto
      • Git clone url_repo: cria uma cópia ou clone de repositórios remotos.

      Push e Pull

      • Git push origin main: enviar o código para o repositório remoto
      • Git pull origin main: obter as modificações de um repositório remoto

      Merge e Rebase

      Utilizados para unir branches.

      • Git merge branch_name: mescla branches com a main criando um novo commit, sem mudar sua estrutura cronológica.
      • Git rebase branch_name: mescla branches com a main colocando todos os commits em ordem cronológica, acarretando numa mudança de estrutura da main.

      Repositórios

      Uma forma de armazenamento de objetos digitais que tem a capacidade de manter e gerenciar material por longos períodos de tempo e prover o acesso apropriado.

      • Git remote add origin url_repo: adicionar um repositório remoto
      • Git clone url_repo: clonar um repositório remoto

      Stash

      Arquiva alterações não commitadas do seu local de trabalho, ou seja, ele volta para o estado do seu último commit guardando as alterações adicionais que você tinha feito. É como se fizesse um backup das modificações dos arquivos

      • Git stash: armazena uma alteração temporária. Arquiva (ou faz o stash) de alterações que você fez na cópia de trabalho (alterações que foram adicionadas ao seu índice, alterações feitas a arquivos que estão no momento rastreados pelo Git) durante um determinado período, para que você possa trabalhar em outra coisa, depois voltar e fazer a reaplicação mais tarde. O stashing é útil quando você precisa alternar com rapidez o contexto e trabalhar em outra coisa, mas está no meio da alteração de código e não está pronto para fazer commit.
      • Git stash --include-untracked: adicionar a opção -u (ou --include-untracked) vai dizer ao git stash para também fazer o stashing dos arquivos não rastreados, porque por padrão, o stashing não se realiza em novos arquivos na cópia de trabalho que ainda não foram preparados e em ignorados.
      • Git stash apply: aplica uima alteração temporária. Aplicar mais uma vez as alterações à cópia de trabalho e manter no stash.
      • Git stash list: listar alterações temporárias. Pode executar o git stash mais da vez para criar múltiplos stashes e, então, usar o git stash list para fazer uma visualização.
      • Git stash clear: limpar uma alteração temporária. Exclui stashes.

      Ignore

      Os arquivos ignorados costumam ser artefatos de desenvolvimento e arquivos gerados por máquina que podem ser derivados da fonte do seu repositório ou que não devem passar por commit. Os arquivos ignorados são rastreados em um arquivo especial chamado .gitignore, que é verificado na origem do seu repositório.

      • .gitignore: nem todos os arquivos precisam ser enviados para o repositório remoto (como arquivos de sistemas).
